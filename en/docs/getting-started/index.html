Getting Started with Kontenum CDN

This guide will help you get up and running quickly with Kontenum, from requesting access to deploying your first CDN site.

Kontenum CDN is designed to accelerate your content delivery across Indonesia with minimal configuration.

Access & Account Creation

Currently, Kontenum is available by invitation only.

To get started:

  1. Visit www.kontenum.com
  2. Request a free trial from the homepage
  3. Our team will contact you shortly
  4. Once approved, we will create your account and provide access to the Console

After your account is ready, you can immediately start creating CDN sites.

Key Concept: What Is a Site?

In Kontenum, a Site represents a CDN configuration similar to a CDN zone or distribution in other providers.

A Site defines:

  • Where your content originates from (Origin)
  • Which domain your users will access (Delivery Domain)
  • How traffic is fetched and delivered through Kontenum‚Äôs CDN

Each Site is deployed independently and can be managed from the Sites section in the Console.

Quick Start: Create Your First Site

You can create your first CDN Site in just a few minutes.

Navigate to:

Console ‚Üí Sites ‚Üí Create New Site

You will see a form similar to the screen below.

Site Configuration Fields

1. Site Name

Give your site a descriptive name to easily identify it later.

Example:

my-website-cdn

2. Origin Type

Choose where your original content is hosted.

  • Domain ‚Äì Use an existing website or server as the origin
  • Storage ‚Äì Use Kontenum Storage as the origin (for static assets)

For most first-time setups, choose Domain.

3. Origin Domain

Enter the domain where your content is currently served from.

Example:

origin.example.com

Kontenum will fetch content from this origin and cache it across our CDN network.

4. Origin Path (Optional)

Specify a sub-path if only part of your origin should be served through the CDN.

Example:

/assets/

Leave this empty to serve content from the root path.

5. Origin Protocol

Select the protocol used to connect to your origin:

  • HTTPS (recommended)
  • HTTP

Make sure this matches your origin server configuration.

6. Delivery Domain

This is the domain your end users will access.

Typically, this is a domain you own and control.

Example:

cdn.example.com
