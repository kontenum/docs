Memulai dengan Kontenum CDN

Panduan ini membantu kamu mulai menggunakan Kontenum dengan cepat, dari request akses sampai deploy site CDN pertama kamu.

Kontenum CDN dirancang untuk mempercepat Delivery Content kamu di seluruh Indonesia dengan konfigurasi minimal.

Akses & Pembuatan Akun

Saat ini, Kontenum tersedia hanya melalui invitation. Langkah untuk memulai:

  1. Kunjungi www.kontenum.com
  2. Request free trial dari halaman utama
  3. Tim kami akan menghubungi kamu segera
  4. Setelah disetujui, kami akan memberikan akses ke Console

Setelah akun kamu siap, kamu bisa langsung mulai membuat CDN Site.

Konsep Utama: Apa itu Site?

Di Kontenum, Site adalah konfigurasi CDN, mirip dengan CDN zone atau distribution di provider lain.

Sebuah Site menentukan:

  • Sumber Konten (Origin)
  • Domain yang akan diakses user (Delivery Domain)
  • Cara traffic diambil dan didistribusikan melalui CDN Kontenum

Setiap Site dikelola secara terpisah melalui menu Sites di Console.

Quick Start: Membuat Site Pertama

Kamu bisa membuat Site CDN pertama kamu hanya dalam beberapa menit.

Melalui langkah-langkah berikut:

Console ‚Üí Sites ‚Üí Create New Site

Kamu akan melihat form seperti gambar di bawah ini.

Field Konfigurasi Site

1. Site Name

Nama untuk mengidentifikasi Site.

Contoh:

my-website-cdn

2. Origin Type

Sumber asal konten.

  • Domain ‚Äì Gunakan website atau server yang sudah ada sebagai origin
  • Storage ‚Äì Gunakan Kontenum Storage sebagai origin (untuk aset statis)

Untuk sebagian besar pengaturan pertama kali, pilih Domain.

3. Origin Domain

Domain tempat konten kamu saat ini di-host.

Contoh:

origin.example.com

Kontenum akan mengambil konten dari origin ini dan menyimpannya di cache di seluruh jaringan CDN kami.

4. Origin Path (Opsional)

Tentukan sub-path jika hanya sebagian dari origin yang ingin di-deliver melalui CDN.

Contoh:

/assets/

Kosongkan jika ingin meng-deliver konten dari root path origin.

5. Origin Protocol

Pilih protokol yang digunakan untuk menghubungkan ke origin:

  • HTTPS (disarankan)
  • HTTP

Pastikan ini sesuai dengan konfigurasi server origin.

6. Delivery Domain

Ini adalah domain yang akan diakses oleh end user.

Domain ini harus kamu miliki dan kelola.

Contoh:

cdn.example.com

Deploy the Site

Setelah semua field diisi:

  1. Klik Create Site
  2. Deployment akan dimulai secara otomatis
  3. Proses ini biasanya memakan waktu sekitar 5 menit

Selanjutnya

Setelah site kamu deployed, kamu bisa:

  • Mengatur DNS untuk Delivery Domain
  • Mengaktifkan HTTPS certificate (certificate milik kamu sendiri atau melalui fitur ACME)
  • Memantau traffic dan penggunaan di Metrics Pelajari dokumentasi lainnya untuk mengoptimalkan penggunaan Kontenum CDN.

Jika kamu membutuhkan bantuan, jangan ragu untuk menghubungi tim Support.
