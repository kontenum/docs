Ringkasan Storage

Kontenum menyediakan object storage yang kompatibel dengan S3 dan dirancang untuk bekerja secara seamless dengan CDN Site Anda. Dokumen ini memberikan ringkasan tentang cara kerja Kontenum Storage, kalkulasi billing, dan kapan layanan ini menjadi pilihan yang tepat untuk workload Anda.

Apa itu Kontenum Storage?

Kontenum Storage adalah layanan object storage kompatibel S3 yang terintegrasi dengan Kontenum CDN.

Karakteristik utama:

  • Storage terkait langsung dengan CDN Site Anda (bukan storage standalone)
  • Dirancang sebagai origin storage dalam alur kerja CDN
  • Kompatibel dengan Amazon S3 API, SDK, dan tooling

Jika Anda pernah menggunakan Amazon S3 sebelumnya, Kontenum Storage akan terasa familiar.

Aktivasi Storage

Secara default, Storage tidak diaktifkan pada akun baru.

Untuk mulai menggunakan Storage:

  1. Buka bagian Storage di Kontenum Console
  2. Tinjau dan setujui Storage & Pricing Agreement
  3. Storage akan diaktifkan untuk organisasi Anda

Model Bucket

Kontenum menggunakan model bucket yang disederhanakan:

  • 1 bucket per akun organisasi
  • Satu bucket dapat digunakan oleh beberapa Site dalam organisasi yang sama
  • Penamaan dan akses bucket dikelola secara otomatis

Pendekatan ini mengurangi kompleksitas operasional sekaligus tetap mempertahankan kompatibilitas dengan alur kerja berbasis S3.

Harga Storage & Komitmen Minimum

Kontenum menerapkan model harga storage dengan komitmen minimum.

Komitmen Storage Minimum

  • Storage minimum: 100 GB
  • Harga: Rp.200 / GB

Artinya:

  • Jika penggunaan Anda di bawah 100 GB, Anda akan dikenakan biaya untuk 100 GB
  • Jika penggunaan Anda di atas 100 GB, Anda akan dikenakan biaya untuk penggunaan aktual

Contoh Harga

  • Penggunaan: 80 GB Biaya: 100 GB √ó Rp.200 = Rp.20,000

  • Penggunaan: 100 GB Biaya: 100 GB √ó Rp.200 = Rp.20,000

  • Penggunaan: 101 GB Biaya: 101 GB √ó Rp.200 = Rp.20,200

Perhitungan Satuan Storage (Basis-2)

Kontenum menggunakan konversi basis-2 (biner) untuk pengukuran storage:

  • 1 GB = 1.024 MB
  • 1 MB = 1.024 KB
  • 1 KB = 1.024 Byte
  • 1 GB = 1.073.741.824 Byte

Sebagian besar sistem object storage menghitung penggunaan storage aktual dengan cara yang sama.

Perhitungan Penggunaan Bulanan

Penagihan storage dihitung sebagai total penggunaan Storage Aktif dan Storage Terhapus per bulan.

Cara Kerja

  • Penggunaan storage dihitung setiap hari
  • Nilai penggunaan harian diakumulasikan sepanjang bulan
  • Objek yang diunggah di tengah bulan akan ditagih secara proporsional

Contoh

  • Anda mengunggah sebuah objek pada hari ke-17 dari bulan dengan 30 hari
  • Objek tersebut ada selama 13 hari dalam bulan tersebut
  • Anda akan dikenakan biaya untuk 13 hari penggunaan ukuran objek tersebut

Retensi Objek Terhapus

Ketika objek dihapus:

  • Objek terhapus akan disimpan selama 90 hari
  • Storage terhapus tetap dihitung sebagai penggunaan bulanan selama masa retensi

üí° Rekomendasi: Hindari penghapusan objek secara sering. Kontenum Storage dioptimalkan untuk konten jangka panjang, bukan pola create-and-delete yang sering.

Versioning Objek

  • Versioning dinonaktifkan secara default
  • Versioning dapat diaktifkan berdasarkan permintaan

Jika Anda memerlukan:

  • Pemulihan objek
  • Perlindungan dari overwrite tidak sengaja
  • Riwayat versi objek

Silakan hubungi tim support Kontenum untuk mengaktifkan versioning.

Kompatibilitas dengan Amazon S3

Kontenum Storage dirancang agar kompatibel secara API dengan Amazon S3, termasuk:

  • S3 REST API
  • AWS SDK
  • Tool dan library S3 umum

Hal ini memungkinkan Anda untuk:

  • Menggunakan kembali aplikasi yang sudah ada
  • Melakukan migrasi workload dengan perubahan minimal
  • Integrasi mudah dengan CI/CD dan workflow backup

Perbandingan Biaya dengan AWS S3

Dampak Komitmen Minimum Storage

Berbeda dengan AWS S3 Standard, Kontenum menerapkan komitmen minimum storage. Beberapa kelas storage AWS (seperti Glacier) memiliki model retensi serupa, namun S3 Standard tidak.

Contoh Perilaku Biaya (100 TB)

  • Jika 100 TB disimpan selama kurang dari 23 hari:
    • AWS S3 bisa menjadi lebih murah
    • Kontenum bisa menjadi lebih mahal
  • Jika 100 TB disimpan selama lebih dari 23 hari:
    • Kontenum menjadi lebih hemat biaya

Keuntungan Biaya Jangka Panjang

  • Setelah 1 tahun, AWS S3 Standard dapat menjadi ~4√ó lebih mahal dibanding Kontenum
  • Ini belum termasuk biaya tambahan AWS seperti:
    • Bandwidth egress
    • Request API

Ringkasan

  • Data jangka pendek (< 23 hari) ‚Üí AWS S3 mungkin lebih hemat
  • Data jangka panjang (> 23 hari) ‚Üí Kontenum biasanya lebih hemat

Kontenum Storage dioptimalkan untuk CDN origin storage, aset statis, dan use case pengiriman konten jangka panjang.
