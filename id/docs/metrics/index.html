Metrik & Penggunaan

Kontenum menyediakan metrik yang detail untuk membantu anda memahami pola traffic, performa cache, dan perilaku CDN secara keseluruhan di seluruh Site anda. Metrik dibagi menjadi dua halaman utama:

  1. Metrics Overview dan
  2. Metrics Details.

Halaman Metrics Overview

Halaman Metrics Overview memberikan ringkasan tingkat tinggi tentang penggunaan CDN anda untuk bulan dan region yang dipilih.

Ringkasan

  • Sites – Total CDN Site aktif di akun anda
  • Users – Jumlah user dalam organisasi
  • Certificates – Total SSL certificate yang di-upload atau diterbitkan

Ringkasan Metrik Bulanan

Grafik overview menampilkan ringkasan penggunaan dari seluruh Site:

  • Bandwidth – Total traffic yang dilayani oleh CDN
  • Total Requests – Jumlah request HTTP yang diproses
  • Cache Miss Ratio – Persentase request yang diteruskan ke origin
  • HTTP Status Code Distribution – Pembagian kode respons (1xx–5xx)

Batas Ambang Cache Miss Ratio

Garis referensi pada 5% cache miss ratio ditampilkan pada grafik.

  • Cache miss ratio di bawah 5% umumnya dianggap healthy dan menunjukkan caching yang efektif
  • Nilai yang lebih tinggi bisa mengindikasikan konten dinamis, cache header yang tidak lengkap, atau purge yang terlalu sering

Threshold ini disediakan sebagai panduan umum dan bisa berbeda tergantung perilaku aplikasi anda.

Metrics Details

Halaman Metrics Details memberikan insight yang lebih mendalam untuk bulan, jenis metrik, dan region yang dipilih.

anda dapat beralih antara:

  • Bandwidth
  • Requests
  • Cache Miss Ratio
  • HTTP Status Code

Setiap tampilan dilengkapi dengan grafik time-series dan breakdown per Site.

Data Terbaru dan Penundaan Pemrosesan

Data metrik dikumpulkan dari seluruh edge location CDN dan kemudian diagregasi secara terpusat.

  • Data near real-time masih bisa berubah dalam 1 jam terakhir
  • Metrik yang lebih baru dari 1 jam dianggap sebagai running data dan dapat diperbarui seiring masuknya data tambahan dari edge
  • Data yang lebih lama dari 1 jam dianggap sudah final

Pendekatan ini memastikan akurasi lintas region, sekaligus tetap memberikan visibilitas yang cepat.

Metrik Bandwidth

Metrik bandwidth menunjukkan jumlah data yang dikirimkan oleh CDN dari waktu ke waktu.

Unit & Konversi

Semua metrik bandwidth menggunakan konversi basis-10 (desimal):

  • 1 Gbps = 1000 Mbps
  • 1 Mbps = 1000 Kbps

ℹ️ Ini berbeda dengan Storage metrics di Kontenum yang menggunakan konversi basis-2 (biner).

Statistik yang Tersedia

  • Total – Total bandwidth yang digunakan pada periode terpilih
  • Min – Throughput terendah yang teramati
  • 95/5 – Nilai persentil ke-95 (digunakan untuk memahami traffic yang berkelanjutan)
  • Max – Maksimum bandwidth tertinggi

Metrik Request

Metrik request menunjukkan jumlah HTTP request yang ditangani oleh CDN.

Statistik yang Tersedia

  • Total Requests – Jumlah total request
  • Min Requests/sec – request per detik terendah
  • Average Requests/sec – Rata-rata request per detik
  • Max Requests/sec – Request per detik tertinggi

Top Sites

Breakdown per Site menampilkan delivery domain yang menghasilkan traffic paling besar.

Cache Miss Ratio

Cache Miss Ratio merepresentasikan persentase request yang harus diteruskan ke origin server.

Interpretasi

  • Low ratio → konten disajikan secara efisien dari cache
  • High ratio → sering terjadi pengambilan konten dari origin

Pada tampilan detail juga tersedia:

  • Grafik Request to Origin
  • Jumlah miss dan miss ratio per Site

Informasi ini membantu mengidentifikasi Site atau path yang bisa dioptimalkan dengan aturan caching yang lebih baik.

Metrik HTTP Status Code

Tampilan ini menunjukkan distribusi respons berdasarkan kelas HTTP status code:

  • 1xx – Informasi
  • 2xx – Respons berhasil
  • 3xx – Pengalihan
  • 4xx – Kesalahan klien
  • 5xx – Kesalahan server

Apa yang Perlu Diperhatikan

  • Persentase 2xx yang tinggi menandakan traffic yang sehat
  • 4xx bisa mengindikasikan asset tidak ditemukan atau request tidak valid
  • 5xx biasanya menunjukkan masalah pada origin server

Tabel per Site menampilkan distribusi status code secara detail untuk setiap delivery domain.

Praktik Terbaik

  • Pantau cache miss ratio secara rutin dan perhatikan lonjakan mendadak
  • Bandingkan tren bandwidth dan request untuk mendeteksi traffic yang tidak normal
  • Gunakan breakdown per Site untuk mengidentifikasi traffic domain yang tidak normal
  • Tunggu setidaknya 1 jam sebelum menggunakan metrik untuk analisis insiden

Jika anda membutuhkan data metrik mentah, akses API, atau bantuan membaca data metrik, silakan hubungi tim support.
