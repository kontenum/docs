Storage Explorer

Storage Explorer menyediakan antarmuka berbasis web untuk mengelola objek di bucket Kontenum Storage Anda. Fitur ini dirancang agar terasa familiar bagi pengguna Amazon S3 Console, sekaligus terintegrasi erat dengan Kontenum CDN.

Ringkasan

Dengan Storage Explorer, Anda dapat:

  • Menelusuri objek dan folder (prefix) di bucket storage
  • Mengunggah file langsung dari browser
  • Membuat folder
  • Mengunduh objek
  • Menghapus file atau folder
  • Melihat hasil operasi dan status upload secara real-time

Kemampuan Manajemen Objek

Menggunakan Storage Explorer, Anda dapat:

  • Menelusuri folder dan objek
  • Mengunggah objek baru
  • Mengunduh objek yang sudah ada
  • Menghapus objek (soft delete, mengikuti kebijakan retensi)
  • Melihat metadata objek, seperti:
    • Ukuran objek
    • Waktu terakhir dimodifikasi
    • Path storage / nama key

⚠️ Objek yang dihapus akan disimpan selama 90 hari dan tetap dihitung sebagai penggunaan storage selama masa retensi tersebut.

Versioning

  • Versioning objek dinonaktifkan secara default.
  • Jika versioning tidak diaktifkan:
    • Mengunggah file dengan key yang sama akan menimpa objek yang sudah ada.
  • Jika versioning diaktifkan (berdasarkan permintaan):
    • Beberapa versi dari key objek yang sama akan disimpan.
    • Semua versi dihitung sebagai penggunaan storage aktif.

Visibilitas Penggunaan dan Penagihan

  • Storage Explorer menampilkan ukuran objek secara logis, bukan ukuran billing yang diprorata.
  • Perhitungan penagihan dilakukan secara terpisah berdasarkan:
    • Storage Aktif
    • Storage Terhapus
    • Akumulasi penggunaan harian selama satu bulan

Untuk detail perhitungan penagihan, silakan lihat dokumentasi Utilisasi Storage.

Navigasi File dan Folder

  • Objek ditampilkan dalam hierarki bergaya folder berdasarkan prefix S3.
  • Navigasi breadcrumb memungkinkan perpindahan cepat antar folder.
  • Folder bersifat fisik (prefix) dan dapat direpresentasikan sebagai objek berukuran nol byte yang diakhiri dengan /.

Unggah Berkas

  • Mendukung drag-and-drop dan pemilihan file manual.
  • Ukuran maksimum upload: 500 MB per file.
  • Beberapa file dapat diunggah secara paralel.
  • Progres dan hasil upload ditampilkan untuk setiap file.
  • Nama file divalidasi untuk mencegah karakter tidak valid dan path yang tidak aman.

Buat Folder

  • Folder dapat dibuat langsung dari UI.
  • Nama folder divalidasi agar kompatibel dengan S3.
  • Folder yang baru dibuat dapat langsung diakses.

Menghapus Objek

  • File dan folder dapat dihapus satu per satu atau secara massal.
  • Menghapus folder akan menghapus semua objek di bawah prefix yang dipilih.
  • Diperlukan langkah konfirmasi sebelum penghapusan.
  • Objek yang dihapus mengikuti kebijakan retensi objek terhapus selama 90 hari.

Log Aksi

  • Menampilkan hasil operasi unggah dan hapus.
  • Menyertakan pesan sukses dan kesalahan untuk transparansi.
  • Berguna untuk mendiagnosis kegagalan parsial selama operasi massal.

Model Keamanan dan Akses

  • Storage Explorer menggunakan temporary presigned URLs untuk unggah dan unduh.
  • Semua operasi mengikuti perilaku dan model izin yang sama seperti Amazon S3–compatible APIs.

Praktik Terbaik

  • Hindari siklus hapus dan unggah ulang yang sering untuk meminimalkan akumulasi storage terhapus.
  • Gunakan konvensi penamaan objek yang konsisten untuk mencegah penimpaan yang tidak disengaja.
  • Untuk otomatisasi atau operasi skala besar, lebih baik menggunakan S3-compatible APIs daripada tindakan manual melalui UI.
